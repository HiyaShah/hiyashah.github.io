Part 1 - Fun with Filters

1.1 Convolutions from Scratch

In this part, I implemented 2D convolution from scratch with both 4-loops and 2-loops in numpy. In the 4-loop implementation, I iterate through every output pixel and every kernel filter element. In the 2-loop implementation, I used numpy vectorized operations by extracting image patches. Please see below for boundary/runtime comparisons between my 4-loop and 2-loop implementations and scipy’s convolve2d implementation.

class Convolution:
	# I also imported and used time to compare wall clock time of my implementation vs scipy's
	def __init__(self, img, kernel):
	        self.img = img 
	        self.kernel = kernel
	        self.kernel_flipped = self.kernel[::-1, ::-1]
	        self.height, self.width = self.img.shape 
	        self.kernel_height, self.kernel_width = self.kernel.shape 
	        self.padding_height, self.padding_width = self.kernel_height//2, self.kernel_width//2
	        self.padded_img = np.pad(self.img, ((self.padding_height, self.padding_height), (self.padding_width, self.padding_width)), mode='constant') 
	    
	def convolution_2_loops(self):
	        img_output = np.zeros((self.height, self.width))
	        for height_i in range(self.height):
	            for width_i in range(self.width):
	                patch_under_kernel = self.padded_img[height_i:height_i + self.kernel_height, width_i:width_i + self.kernel_width]
	                img_output[height_i, width_i] = np.sum(patch_under_kernel * self.kernel_flipped)
	        return img_output
	def convolution_4_loops(self):
	        img_output = np.zeros((self.height, self.width))
	        for height_i in range(self.height):
	            for width_i in range(self.width):
	                sum = 0
	                for kernel_height_i in range(self.kernel_height):
	                    for kernel_width_i in range(self.kernel_width):
	                        sum += self.padded_img[height_i + kernel_height_i, width_i + kernel_width_i] * self.kernel_flipped[kernel_height_i, kernel_width_i]
	                img_output[height_i, width_i] = sum
	        return img_output

1.3 Derivative of Gaussian (DoG) Filter

I created the 2D gaussian filter by using cv2.getGaussianKernel() to get the 1d gaussian and taking its outer product as below with default kernel size 9x9 and sigma=1.6. I’ve displayed the DoG_x and DoG_y filters as images below, and they act like oriented edge detectors.

def gaussian_kernel_2d(ksize=9, sigma=1.6):
        gaussian_1d = cv2.getGaussianKernel(ksize, sigma)        
        return (gaussian_1d @ gaussian_1d.T).astype(np.float32)

Multiresolution Blending & Oraple

2.3 Gaussian & Laplacian Stacks

To build the Gaussian stack, I started with the original image and progressively blurred it with Gaussian filters of increasing σ, storing all levels at the same resolution (no downsampling). The Laplacian stack was then computed as the difference between consecutive Gaussian levels, with the last level being the most blurred residual. The final blended Laplacian stack was created by weighting each level of the apple and orange stacks with the corresponding mask, and I collapsed the stack by summing from the coarsest to finest levels to reconstruct the full-res “oraple”.

2.4 Multiresolution Blending

For multiresolution blending, I used the Gaussian and Laplacian stacks from Part 2.3 along with a Gaussian stack of a binary mask to combine two images at diff frequency bands. At each level, I multiplied image1 and image2 Laplacians by the corresponding blurred mask and its complement, summed them to get a blended Laplacian level, and then collapsed (reconstructed) the blended Laplacian stack from coarse to fine. I first tested this on the oraple with vertical step mask and then built some irregular masks for Anne Hathaway/Miranda Priestly, Spiderverse Portal, and Remy’s Cooking Ghost.
