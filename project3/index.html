Part A.2: Recover Homographies

In this part, my goal was to estimate a homography transformation matrix H between an image A to its projective transformation image B which has the same center of projection. To recover this matrix, I needed to label a set of N≥4 feature correspondence points im1_pts (xi,yi)(x_i, y_i) (xi​,yi​)﻿ and im2_pts (xi′,yi′)(x_i', y_i')(xi′​,yi′​)﻿ such that the transformation is p′≈Hpp'\approx Hpp′≈Hp﻿ . While H would ideally find an exact transformation between the points in the images, due to noise when I select correspondence points manually, I need to select more than 4 corresponding points, and the above transformation equation is an overdetermined system from which I can recover H via least squares. Although H is a 3x3 matrix, there are 8 unknowns because we fix the lower right to be 1.

I wrote my own mouse-click interface tool using matplotlib ginput, which allowed me to select 8 points on both the left and right images to establish the point correspondences between them.

Given my n correspondence point pairs, I now construct the system of linear equations Ah=bAh = bAh=b﻿, in which AAA﻿ is a 2n x 8 matrix, h is the solution vector containing the 8 unknowns of the homography matrix, and b contains the target coordinates in the other image.

Part A.3 Warp the ImagesNearets

I had to resize the images to ~1000px on the longest side to prevent OOM for both Nearest Neighbor Interpolation and Bilinear Interpolation warping methods. While the quality between the two warping methods was similar, I noticed that bilinear warping resulted in a slightly more blurred rectification (ex on the edges of the conference collection book) while the nearest neighbor method produced more jagged edges. For speed, on average, nearest neighbor interpolation warp took 31.224 ms and the slower bilinear interpolation warp took 84.385 ms with overall 2.70x speedup.

For nearest neighbor interpolation warping, for each pixel in the image 2, I mapped its coordinates back to the source image 1 using inverse homography and rounded to the nearest integer because the resulting coordinates usually fall between pixels. For bilinear interpolation warping, I took weighted average of the four surrounding pixels in image 1 based on how close the mapped point was to each one, and I think this resulted in better blending but computationally slower result.

Part A.4 Blend the Images into a Mosaic

To blend the images into a mosaic, my procedure was as follows. I first computed the homography between the left and right images by manually clicking corresponding points that overlapped in both images using my matplotlib ginput correspondence clicking tool. I then computed a bounding box of the projected to determine the dimensions of the mosaic and the translation offset I would need so that all coordinates are positive so that the warped images will fit inside one canvas. Using inverse warping, I warped the images into a common frame using both the nearest neighbor interpolation and bilinear interpolation warping methods, and each warped image produced an alpha mask. I applied a shift matrix and premultiplied it with the H matrix from homography to put the images in the correct positions in the canvas. I then noticed that there were visible seams when I overlapped the two images, so I performed alpha feather blending to resolve this by generating a linear alpha ramp that decreased 1→0 for overlap over image 1 and 0→1 for overlap over image 2, and I blended these using weighted averaging.

Part B.1 Harris Corner Detection / Adaptive Non-Maximal Suppression

I read section 2 of the MOPS paper, and learned that the Harris corner detection for interest points works by finding regions in an image where local intensity changes in both x and y directions significantly by computing Harris structure matrix at each pixel and determining corner strength which is the harmonic mean of eigenvalues and points of interest are the local maxima. In peak_local_max call in get_harris_corners, I set min_distance = 2 (instead of 1) and added threshold_rel = 0.01, resulting in 14510 corners detected. When I had min_distance = 1 as was originally in the provided harris.py, I got 318482 points and the anms run resulted in out-of-memory since it would be an O(N^2) operation, so I had to adjust the number of corners detected in the original harris corner detection.

Adaptive Non-Maximal Suppression (ANMS) reduces the number of interest points. It assigns the corners a suppression radius (or the distance to the nearest point with the higher corner strength) and selects the top 500 points so that the interests points are spatially distributed across the image. The algorithm works as follows. On each interest point by computing the points whose corner strength * c_robust (0.9 in this case) is greater than corner of the interest point and computes the closest distance of the stronger interest points to this interest point we are currently considering, picks top k (500) with the largest suppression radius, ensuring good coverage of the image.

Part B.2 Feature Descriptor Extraction

For each interest point from ANMS, I take a 40x40 pixel region around it and downsample to 8x8 grid which smooths out the high frequency noise, making it overall more tolerant to small lighting variations. I then bias and gain normalize each patch by subtracting its mean intensity and dividing by std. deviation.

Part B.3 Feature Matching

By feature matching, I compare feature descriptors between different interest points in different images by comparing their feature descriptors so that each feature in one image is compared to all others in the other image using squared Euclidean distance similarity measure. To this this, I apply Lowe ratio test such that for a given descriptor, I find the two nearest neighbors in the other image and the match is accepted only if their ratio is below the threshold I chose (0.65), and we want to have the best match be significantly better than second best.

Part B.4 RANSAC

I utilized RANSAC (Random Sampling and Consensus) to find the robust set of pairs of correspondences to estimate homographies. In each iteration, I

  1. Sample a 4-correspondence set {(x⃗0,x⃗0′), (x⃗1,x⃗1′), (x⃗2,x⃗2′), (x⃗3,x⃗3′)}\{ (\vec{x}_0, \vec{x}'_0),\, (\vec{x}_1, \vec{x}'_1),\, (\vec{x}_2, \vec{x}'_2),\, (\vec{x}_3, \vec{x}'_3) \}{(x0​,x0′​),(x1​,x1′​),(x2​,x2′​),(x3​,x3′​)}﻿ between the images A and B
  1. Compute the homography matrix H using the points above
  1. Apply H to all of the matches and and compute the reprojection error for each pair ∥x⃗k′−Hx⃗k∥22\|\vec{x}'_k - H\vec{x}_k\|_2^2∥xk′​−Hxk​∥22​﻿ which is the distance between the transformed source point and matched point in target image. I considered the points below a threshold ϵ\epsilonϵ﻿ I set as inliers.
  1. I repeated the above steps a lot of times to get the largest set of inliers and then applied least squares to them to get a more robust H estimated since there are no longer large outliers.
